# BeautyBook Pro - Magic Link Authentication Guide

## 🔐 Overview

The operator login is now **protected by magic links** generated by the super admin. This ensures that:
- Only authorized staff can access the login page
- Login is revoked by invalidating the magic link
- Each access is traceable and auditable

## ✅ How It Works

### For Users (Staff)

1. **Receive Magic Link**: Super admin sends you a link via email (e.g., above)
2. **Click Link**: Click the link - it automatically opens the login page
3. **Enter Password**: Your email is pre-filled, just enter your password
4. **Access Dashboard**: After login, you can access the operator dashboard

### For Developers/Testing

**To test the magic link system:**

1. Copy one of the test links from above
2. Paste it in your browser address bar
3. You'll be redirected to the login page with the email pre-filled
4. Enter the password and sign in

**If you try to access `/operators/login` directly without a token:**
- You'll see an "Access Restricted" message
- After 3 seconds, you'll be redirected to the home page

### Magic Link Flow

```
1. Super Admin generates link (future feature)
   ↓
2. Link contains: /operators/login?token=ENCODED_TOKEN
   ↓
3. User clicks link
   ↓
4. Login page validates token
   ↓
5. If valid: Show login form (email pre-filled)
   ↓
6. If invalid: Show "Access Restricted" & redirect
```

### Token Structure

The token is base64 encoded with this format:
```
email|timestamp|random_string
```

Example:
```
admin@beautybook.pro|1745600000|abcdef123
```

## 🎯 Key Features

✅ **On-Demand Access**: Links can be created whenever a new operator joins
✅ **Time-Limited**: Tokens expire after 24 hours
✅ **No Public Access**: Login page is completely hidden from public sight
✅ **Email Pre-fill**: User's email is automatically filled from the token
✅ **Audit Trail**: Every link can be logged for security tracking
✅ **Revocable**: Super admin can disable links anytime

## 🔒 Security Notes

### Current Implementation (Development)

⚠️ The temporary magic links above are **hardcoded for testing only**.

### Production Requirements

Before deploying to production:

1. **Generate Links Backend**: Create links server-side, never in frontend
2. **Store in Database**: Save tokens with:
   - Hashed token value
   - Operator email
   - Creation timestamp
   - Expiration timestamp
   - Status (active/revoked)

3. **Validate on Login**:
```javascript
// Example backend validation
const token = await db.query(
  'SELECT * FROM magic_links WHERE token_hash = ? AND status = "active"'
)

if (!token || token.expires_at < now) {
  return { error: 'Invalid or expired token' }
}
```

4. **After Successful Login**:
   - Mark token as used/revoked
   - Create session/JWT token for user
   - Clear magic link

5. **Email Delivery**:
   - Send links via email with expiration notice
   - Don't log links in plain text
   - Use secure email delivery (TLS/SSL)

## 🐛 Troubleshooting

### "Access Restricted" Message

**Causes:**
- No token provided in URL
- Token is expired (older than 24 hours)
- Token is invalid/malformed
- Token doesn't exist in database (future)

**Solution:**
- Request a new magic link from super admin
- Check the URL for typos
- Copy the full link including the `?token=...` part

### Email Not Showing in Login Form

**Cause:** Token might be invalid

**Solution:**
- Verify the magic link is correct
- Try a different test link
- Check browser console for errors

### Can't Access Login Page Directly

**This is intentional!** The login page is only accessible via magic link for security. You cannot visit `/operators/login` directly.
